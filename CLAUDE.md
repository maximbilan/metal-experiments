# CLAUDE.md

Guidance for Claude Code when working in this repo.

## What this repo is

`metal-experiments` is a collection of small, **self-contained visual effects for
iOS**. Each effect is one folder that renders entirely through **a single SwiftUI
`colorEffect` modifier backed by a single `[[stitchable]]` Metal shader** — no
render pipeline, no textures, no assets. Everything is procedural.

Existing effects to use as reference: `singularity/`, `water/`, `martian-sand/`,
`snow/`. When adding an effect, **copy the most recent one and adapt it** rather
than starting from scratch — `snow/` and `martian-sand/` are the cleanest templates.

## Folder layout (every effect is identical)

```
<effect>/
├── README.md              # what it is + how to drop it into an app
├── project.yml            # xcodegen spec for the standalone demo app
├── screenshot.jpg         # full-res preview (1206×2622, from the simulator)
├── demo.mp4               # ~9–12s screen capture
└── Sources/
    ├── <Effect>.metal     # the shader
    └── <Effect>App.swift  # the SwiftUI app that hosts it
```

`<Effect>.xcodeproj` is **generated by xcodegen and gitignored** — never commit it.
The tracked files are exactly the six above.

## Naming conventions

| Thing | Convention | Example (Snow) |
|---|---|---|
| Folder | lowercase, hyphenated | `snow/`, `martian-sand/` |
| Effect / type name | PascalCase | `Snow`, `MartianSand` |
| Source files | `<Effect>.metal`, `<Effect>App.swift` | `Snow.metal`, `SnowApp.swift` |
| Shader function | camelCase | `snowfall`, `martianSand` |
| Bundle id | `com.max.<lowercase>` | `com.max.snow` |
| xcodegen `name` | PascalCase (== scheme) | `Snow` |

## The shader (`Sources/<Effect>.metal`)

- `#include <metal_stdlib>` / `using namespace metal;`
- The entry point is a `colorEffect` function with this **exact** signature (the
  first two params are supplied by SwiftUI; the rest are your uniforms):

  ```metal
  [[ stitchable ]] half4 <fn>(float2 position, half4 color, float2 size,
                              float time, float2 touch, float touchActive) {
      ...
      return half4(half3(col), 1.0);
  }
  ```

- **Shared helpers** — copy these verbatim so the math reads consistently across
  the repo. The core set every effect uses: `hash21(p)`, `vnoise(p)` (value noise),
  `fbm(p)` (4-octave). Add `rot(p, a)` when you rotate/swirl and `hash22(p)` when
  you need 2D randomness. Mark them all `static`.
- **Coordinate conventions:**
  - `float mn = min(size.x, size.y);`
  - `float2 uv = position / mn;` — aspect-correct space for round shapes / world math.
  - `float2 p = position / size;` — 0..1 screen space, for gradients/vignette.
- **Touch interaction** is expected. `touch` is in pixels (divide by `mn`),
  `touchActive` is 0/1. The established pattern is a localized `exp(-dist*k)`
  falloff around the finger — e.g. `snow`/`martian-sand` swirl the field with
  `rot()` and kick up extra particles.
- **Finish the frame** the way the others do: fine `hash21` grain, a radial
  vignette (`col *= 1.0 - k*dot(sv, sv)`), and a filmic tonemap
  (`col = 1.0 - exp(-col * exposure)`). Keep it in a tasteful, non-neon palette.
- Keep it performant per-pixel: bounded loops, no dynamic branching on
  per-pixel data where avoidable. Fragment-only; there is no vertex stage.

## The app (`Sources/<Effect>App.swift`)

Boilerplate is identical across effects — only the shader name, the initial
`center`, and the doc comment change:

```swift
import SwiftUI

@main
struct <Effect>App: App {
    var body: some Scene { WindowGroup { ContentView() } }
}

struct ContentView: View {
    @State private var touch: CGPoint?
    @State private var isTouching = false
    private let start = Date()

    var body: some View {
        GeometryReader { geo in
            TimelineView(.animation) { timeline in
                let t = timeline.date.timeIntervalSince(start)
                let center = CGPoint(x: geo.size.width / 2, y: geo.size.height * 0.45)
                Rectangle()
                    .colorEffect(ShaderLibrary.<fn>(
                        .float2(geo.size),
                        .float(Float(t)),
                        .float2(touch ?? center),
                        .float(isTouching ? 1 : 0)
                    ))
            }
            .gesture(
                DragGesture(minimumDistance: 0)
                    .onChanged { touch = $0.location; isTouching = true }
                    .onEnded { _ in isTouching = false }
            )
        }
        .ignoresSafeArea()
        .background(.black)
        .statusBarHidden()
    }
}
```

> SourceKit may flag `@main` / `statusBarHidden` before the `.xcodeproj` exists
> (it defaults to a macOS context). These clear once xcodegen generates the iOS
> target — ignore them.

## `project.yml`

```yaml
name: <Effect>
options:
  bundleIdPrefix: com.max
targets:
  <Effect>:
    type: application
    platform: iOS
    deploymentTarget: "17.0"
    sources: [Sources]
    settings:
      GENERATE_INFOPLIST_FILE: YES
      INFOPLIST_KEY_UILaunchScreen_Generation: YES
      PRODUCT_BUNDLE_IDENTIFIER: com.max.<lowercase>
```

Requires **iOS 17+** (the SwiftUI `Shader` API). Deployment target is `"17.0"`.

## Build, run, and capture assets

```sh
cd <effect>
xcodegen generate            # brew install xcodegen
```

Then build/run on a simulator (via Xcode, or the XcodeBuildMCP tools:
`session_set_defaults` → `build_run_sim`). To capture the committed assets from a
booted simulator (`$SIM` = its UDID):

```sh
# screenshot.jpg — full device resolution, matches the other samples
xcrun simctl io "$SIM" screenshot /tmp/shot.png
sips -s format jpeg -s formatOptions 90 /tmp/shot.png --out screenshot.jpg

# demo.mp4 — record with record_sim_video (or `xcrun simctl io "$SIM" recordVideo`),
# then trim to ~9–12s and compress to ~3–5 Mbps to match the repo:
ffmpeg -y -ss 4 -i /tmp/raw.mp4 -t 12 \
  -c:v libx264 -pix_fmt yuv420p -crf 26 -preset slow -an \
  -movflags +faststart demo.mp4
```

Target sizes: screenshot ≈ 1206×2622, demo a few MB. Verify the frame actually
shows the effect (not a launch/black screen) before committing.

## When adding an effect — checklist

1. Create `<effect>/Sources/`, write `<Effect>.metal` and `<Effect>App.swift`.
2. Write `<effect>/project.yml` and `<effect>/README.md` (copy an existing
   README and adapt the "How it works" / snippet).
3. `xcodegen generate`, build & run on a simulator, confirm it looks right and
   the touch interaction works.
4. Capture `screenshot.jpg` and `demo.mp4` as above.
5. Add a row to the **top-level `README.md`** table (Preview | Effect | Description),
   following the existing format exactly.
6. Confirm `git status` shows only the six tracked files + the README change —
   the `.xcodeproj` must stay ignored.

## Git / PR workflow

- **Never commit or open a PR without the user's approval.**
- Branch name = the effect's folder name (`snow`, `water`, `martian-sand`).
- Commit subject: `Add <Effect>: <short description> shader for SwiftUI`.
- End commit messages with the trailer:
  `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`
- Open a PR against `main`; every change in this repo lands through a PR.
